# Portfolio 2

Documentation of the project layout and what lives in each folder.

## Project structure

- `src/` – React (Vite + Tailwind) source code for the portfolio experience.
  - `components/` – Page sections and shared UI (Hero, Navbar, showcase cards, utility pages like network checker/updater).
  - `pages/` – Route entry points for the main site and standalone helper pages.
  - `data/` – Metadata and content used to populate sections (JSON + TS helpers).
  - `hooks/` – Custom React hooks for animation timing, viewport detection, carousel behavior, and event handling.
  - `utils/` – Utility helpers (class names, throttling, navigation/meta builders).
  - `constants.tsx`, `types.ts`, `index.css`, `App.tsx` – App shell, shared types, styling, and constants.
- `public/` – Static assets served as-is.
  - Favicons, robots, theme init script, and social card images at the root of `public/`.
- `dist/` – Production build output generated by `npm run build`.
- `READMEs/` – Additional documentation artifacts (usage and security notes from the template).
- `node_modules/` – Installed dependencies (generated by `npm install`).
